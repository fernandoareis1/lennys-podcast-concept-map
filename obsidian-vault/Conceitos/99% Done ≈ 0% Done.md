# 99% Done ≈ 0% Done

**Fluxo:** [[04 - Design & UX]] · **Tema:** [[Design — Experiência & fricção]] · **Camada:** L3
**Tipo:** Mindset · **Fontes:** [[Dmitry Zlokazov]]

## Ideia central
Um produto 99% pronto é, na prática, **mais próximo de 0% do que de 100%**. Pormenores de qualidade, UX e polish não são detalhes finais — eles são o produto. A Revolut nunca lança algo que não seja genuinamente "wow", mesmo que isso signifique reduzir drasticamente o escopo de funcionalidades.

**O problema da ambiguidade de causa:**
Se você lança um MVP de qualidade ruim e ele não ganha tração, você não sabe se o problema é:
- A ideia em si está errada
- O produto simplesmente não é bom o suficiente

Ao sempre lançar com altíssima qualidade, você remove essa incerteza. Se não ganhar tração, é porque a ideia estava errada — e essa é uma conclusão válida.

**Trade-off explícito:**
- Corte funcionalidades para lançar menor
- Mas nunca comprometa qualidade, UX e estética no que foi decidido lançar

## Como aplicar
1. Antes de lançar, pergunte: "Isso é genuinamente wow? Ou é 99% — que na prática é 0%?"
2. Prefira lançar metade das features com qualidade total a todas as features com qualidade parcial
3. Use polish como critério de go/no-go — não apenas "funciona", mas "encanta?"
4. Lance primeiro para grupo pequeno (alpha cohort) e só scale depois que métricas de retenção forem sólidas
5. Founders/líderes seniores devem revisar todo UI antes do lançamento — não como gargalo, mas como padrão de barra alta
