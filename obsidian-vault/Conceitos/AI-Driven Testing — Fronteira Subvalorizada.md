# AI-Driven Testing — Fronteira Subvalorizada
**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Inbal Shani]]

## Ideia central
Toda a atenção de IA no desenvolvimento de software foi para **geração de código**. O que está sendo ignorado: **testing**.

A lógica é simples: se IA vai gerar muito mais código, muito mais rápido, por muito mais pessoas (inclusive não-desenvolvedores), a quantidade de software no mundo vai explodir. A única forma de validar todo esse código é com testing — e hoje testing exige especialistas humanos para cada tipo (unit, integração, carga, performance, segurança, penetração).

**Os tipos de teste que AI poderia gerar automaticamente:**
- Unit testing e functional testing
- Integration testing
- Load testing e performance testing
- Infrastructure testing
- Security testing e penetration testing

Hoje a maioria desses testes não é feita com rigor porque exige muito trabalho humano especializado. Com AI gerando suites de teste completas, o gargalo muda: de "quem escreve os testes?" para "o que precisa ser testado e por quê?"

**Implicação estratégica:**
Para qualquer empresa que está construindo produtos de AI para desenvolvedores, ou que usa AI para gerar código internamente: a oportunidade de diferenciação e de valor está no testing assistido por AI, não (só) na geração de código.
