# Armar o champion no deal B2B
**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Posicionamento & diferenciação]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[April Dunford]]

## Ideia central
Em compras B2B típicas, há 5 a 7 pessoas envolvidas, mas apenas uma é o **champion** — a pessoa designada para encontrar e avaliar a solução. O champion pode fazer o deal acontecer; as outras pessoas não conseguem fazer o deal acontecer, mas podem matá-lo de mil formas (IT, financeiro, jurídico, usuários finais). A estratégia correta: **conquiste o champion primeiro** com seu pitch de posicionamento; depois, **arme o champion** para lidar com todos os outros stakeholders. Você tem o conhecimento — o champion não. Seja o guia deles no processo de compra.

## Como aplicar
1. **Identifique e foque no champion**: é quem fez a pesquisa inicial, criou a shortlist, está nas primeiras calls. Seu pitch de valor precisa funcionar para esta pessoa.
2. **Mapeie as objeções antecipadamente**: depois de 10-15 deals, você sabe exatamente o que IT vai perguntar, o que o financeiro vai questionar, o que os usuários finais vão reclamar. Use esse conhecimento.
3. **Arme o champion na própria call**: "IT vai perguntar sobre segurança — aqui está o que você vai dizer. Seu chefe vai querer ROI — aqui está nossa calculadora para isso."
4. **Processe de compra como serviço**: se o champion nunca comprou software como o seu (geralmente é a primeira vez), explique o processo de compra típico para eles: "Provavelmente vai para procurement. Eles vão pedir X. Você precisará de Y."
5. **Não tente converter os bloqueadores**: IT não está comprando valor, está verificando compliance. Financeiro não está comprando produto, está verificando orçamento. Trate essas interações como gestão de objeções, não como pitch de venda.

## Insights por autor
### [[April Dunford]]
- "Os outros stakeholders não conseguem fazer o deal acontecer, mas podem matá-lo de mil jeitos. Então sua missão, uma vez que o deal está andando, é armar o champion para lidar com todos esses grupos."
- "Metade do tempo você está fazendo deals e eles nem sabem qual é o processo de compra. Se você é um bom parceiro, você diz: 'Vai para procurement. Eles vão fazer isso. E você vai precisar entregar isso.'"
- Champion task: A pergunta que o champion está tentando responder não é só "qual software é melhor?" — é "como faço uma decisão sem ser demitido?" Você tem que ajudá-los a responder as duas perguntas.
- Qualificação acontece antes: discovery no setup do pitch deve identificar se este é o champion certo e se a fit existe. Se não houver fit, é disqualify — não uma objeção para superar.
- Postura certa: "confiança calma" — você sabe quem deveria escolher você e não vai atrás de quem não deveria.
