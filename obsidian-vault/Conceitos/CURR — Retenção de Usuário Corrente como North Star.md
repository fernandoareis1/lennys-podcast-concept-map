# CURR — Retenção de Usuário Corrente como North Star

**Fluxo:** [[07 - Ativação & Retenção]] · **Tema:** [[Retenção — Retenção & churn]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Jackson Shuttleworth]]

## Ideia central
CURR (Current User Retention Rate) mede a taxa com que usuários ativos (não novos, não ressuscitados) retornam no dia seguinte. O Duolingo descobriu empiricamente que, dentre todas as métricas disponíveis, uma mudança percentual no CURR tem o maior impacto sobre o DAU — maior que novos usuários, maior que ressurreição. Isso torna o CURR o North Star operacional da equipe de retenção, e o streak é a feature mais eficaz em movê-lo.

## Como aplicar
- Segmente sua retenção em três grupos: novos usuários, usuários ressuscitados, usuários correntes.
- Meça separadamente a retenção D+1 de cada grupo.
- Calcule o impacto de uma variação de 1pp em cada — em geral, usuários correntes têm maior base e maior impacto.
- Priorize features que movem CURR antes de features que movem novos usuários.
- Use esse framework para justificar investimento em features de engajamento que parecem "de baixo impacto" mas têm grande base multiplicada.
