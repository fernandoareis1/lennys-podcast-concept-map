# Canonical Doc

**Fluxo:** [[09 - Liderança & Times]] · **Tema:** [[Liderança — Colaboração & comunicação]] · **Camada:** L3
**Tipo:** tecnica · **Fontes:** [[Naomi Gleit]]

## Ideia central
Para cada projeto deve existir **um único documento canônico** que qualquer pessoa pode encontrar e que contém tudo o que ela precisa saber. Centenas de docs podem existir, mas todos são satélites do canônico. Sem isso, cinco pessoas dão cinco respostas diferentes à mesma pergunta — inaceitável.

## Como aplicar
1. Crie o doc canônico ao começar um projeto; todos devem saber onde ele está.
2. **Conteúdo mínimo do doc canônico:**
   - Work streams (áreas de trabalho discretas)
   - Single-threaded owner (STO) de cada work stream — e de cada sub-work-stream
   - Processo de trabalho: reunião canônica, lista de e-mail canônica, chat canônico
   - Nomenclatura canônica (glossário)
   - Visual/framework canônico que representa o projeto
3. Linke **todos os outros docs** a partir do canônico — o canônico é o índice, não o repositório completo.
4. **Recursividade obrigatória:** se um work stream é grande o suficiente para ter sub-work-streams, o canônico do work-stream pai aponta para o canônico do filho.
5. Ao entrar em um projeto novo, a primeira pergunta é: "Qual é o canonical doc?" — e deve haver uma resposta imediata.

## Insights por autor
### [[Naomi Gleit]]
- "Canonical" se tornou tão associado a Naomi que colegas da Meta fizeram pôsteres de "Canonical, canonical, canonical" para dar a ela em datas comemorativas.
- Colegas a descrevem como "lendária por seus canonical documents" (Charles Porch, VP Instagram).
- O doc canônico resolveu o problema de entrar em projetos no meio e ter de perguntar para 5 pessoas, obtendo 5 respostas diferentes.
