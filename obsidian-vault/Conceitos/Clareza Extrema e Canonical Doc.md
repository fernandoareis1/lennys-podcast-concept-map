# Clareza Extrema e Canonical Doc

**Fluxo:** [[09 - Liderança & Times]] · **Tema:** [[Liderança — Colaboração & comunicação]] · **Camada:** L3
**Tipo:** framework · **Fontes:** [[Naomi Gleit]]

## Ideia central
**Extreme Clarity** significa que todos compartilham o mesmo entendimento dos fatos — não necessariamente concordam, mas partem da mesma base. A tática central é o **canonical doc**: um único documento de referência por projeto que toda a equipe sabe onde está, que lista work streams, owners (single-threaded), processos, nomenclaturas e visuais. Sem canonical, perguntas simples geram 5 respostas diferentes — desperdício puro.

## Como aplicar
**Canonical Doc — elementos obrigatórios:**
1. **Work streams** e seus **single-threaded owners** — um dono claro por work stream (e sub-work stream), sem ambiguidade de accountability.
2. **Nomenclatura canônica** — glossário explícito dos termos-chave; pessoas usam as mesmas palavras para as mesmas coisas.
3. **Processo canônico** — como o time trabalha junto: reuniões fixas, lista de e-mail, chat canônico (não crie 10 grupos com permutações diferentes).
4. **Visuais canônicos** — toda decisão tem uma representação visual; na reunião, edite o slide em tempo real para capturar o que foi decidido (opção escolhida, próximos passos numerados).

**Táticas adicionais:**
- **Listas numeradas, nunca com bullets** — só com numeração você pode referenciar "o ponto 3"; bullets são impossíveis de citar.
- **Traffic light para decisões** — opções nas linhas, critérios de avaliação (legal, produto, privacidade, UX) nas colunas; células coloridas vermelho/amarelo/verde; a recomendação fica clara visualmente.
- **Pre-read 24h antes** da reunião; pós-reunião, reply-all no convite com decisões e próximos passos em até 24h.
- Quando entrar em projeto já em andamento, pergunte pelo canonical doc; se não existir, criá-lo é o primeiro passo.

## Insights por autor
### [[Naomi Gleit]]
- "Extreme clarity não significa que todos concordam — significa que todos entendem os mesmos fatos. Podemos discordar, mas partimos da mesma base."
- "Eu nunca uso bullet points. Com lista numerada dá para referenciar o ponto 2. Com bullet, você fica dizendo 'o terceiro bullet dois acima do segundo'."
- "Para qualquer projeto, a pergunta imediata é: onde está o canonical doc? Todo mundo deve saber."
- O processo de criar o canonical doc frequentemente revela que o problema real é de pessoas ou processo (80% dos casos), não de estratégia.
