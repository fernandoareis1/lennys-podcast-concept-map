# Clareza sobre Engenhosidade — Design Legível
**Fluxo:** [[04 - Design & UX]] · **Tema:** [[Design — Experiência & fricção]] · **Camada:** L3
**Tipo:** Princípio · **Fontes:** [[Robby Stein]]

## Ideia central
Quando algo é uma **convenção estabelecida** — uma câmera, um feed, uma forma de navegar — reinventá-la para parecer criativa quase sempre prejudica o produto. As pessoas **pensam espacialmente**: elas têm modelos mentais de onde elementos aparecem e o que fazem. Subverter esses modelos sem razão clara é pagar o custo do aprendizado do usuário sem dar retorno.

A regra: **seja coerente mas diferente**. Apóie-se nos standards para que o usuário saiba como operar, mas adicione sua diferenciação naquilo que realmente importa para o seu produto.

Exemplos concretos do princípio:
- **Close Friends ring**: o anel verde do Close Friends estava DENTRO da história (clever, secreto). Usuários não entendiam o que aquela história tinha de especial. Movê-lo para FORA — visível no tray — aumentou o engajamento drasticamente. A clareza bateu a sutileza.
- **Ícones**: equipes reinventam câmera como "ícone que é câmera mas também tem IA e tem pontos que conectam ao outro produto." O usuário vê uma câmera. Só uma câmera.
- **Don Norman e as portas**: uma porta com alças simétricas em ambos os lados não comunica se você deve puxar ou empurrar. O design elegante destruiu a informação que o usuário precisa.

## Como aplicar
1. **Identifique o que é standard vs o que é sua diferenciação**: qual parte do seu produto o usuário já conhece de outros contextos? Qual parte é sua contribuição única?
2. **Nos standards, lean in**: use os ícones conhecidos, as posições esperadas, os fluxos convencionais. O usuário não precisa aprender.
3. **Reserve a diferenciação para o que importa**: a novidade vai para o conteúdo, a funcionalidade diferenciada, não para o wrapper de navegação.
4. **Teste pelo modelo mental espacial**: onde o usuário acha que aquele elemento estará? O que ele acha que aquele ícone faz? Se errar, você perdeu.
5. **"Coherent but different"**: o novo produto deve sentir-se como parte do sistema (coerente) mas claramente distinto (para expectativas não colidirem).

## Insights por autor
### [[Robby Stein]]
- "Design for clarity instead of cleverness. If something is a standard and people understand it, if you lean into it you're going to get so much leverage than if you reinvent it."
- "People think spatially. So if you have a feed and you have holes with pictures, they expect those holes to do things. If you make one of those holes operate differently, you have to be crystal clear about it."
- Sobre o green ring: "We were being cute. We thought it was this secret story and if you open it, you see it. It just was not clear to people. So we put the green ring on the outside so users would see it in the tray and be like: ooh, what's that little green guy?"
- "It needs to feel coherent but different. Stories has similar aesthetic, uses your camera roll, works in DM — but it has a different primitive."
- "Don Norman's book — he has an incredible chapter about doors. After all these years, people still don't know if you should pull or push a door, because the designer built the most beautiful symmetric handles on each side."
