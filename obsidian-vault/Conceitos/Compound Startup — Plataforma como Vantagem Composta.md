# Compound Startup — Plataforma como Vantagem Composta

**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Jeremy Henrickson]]

## Ideia central
Um compound startup é uma coleção de negócios independentes que compartilham uma única plataforma de dados — um único sistema de registro. O insight original de Parker Conrad na Rippling: produtos de HR, payroll, benefícios, IT e compliance existem como empresas multibilionárias separadas porque nenhuma delas tem um sistema de registro centralizado. Quando todos esses dados vivem no mesmo lugar, você consegue fazer coisas literalmente impossíveis para qualquer concorrente — como responder automaticamente "quem é o gestor dessa pessoa?" em qualquer produto. Cada novo produto tem activation energy alta (você precisa construir o primeiro produto para ter a plataforma), mas depois de construída, cada novo vertical fica mais barato, mais rápido e mais diferenciado porque herda a plataforma. Modelo de execução: time de 4–6 pessoas (1 engenheiro-founder, 1 designer, 2–3 engenheiros) lança novo produto em 6–9 meses, depois escala. Design para o caso mais complexo primeiro — não o MVP mais simples — pois decisões arquiteturais iniciais determinam o teto do produto.

## Como aplicar
1. Antes de construir mais features no produto existente, pergunte: "Qual plataforma unificada de dados tornaria um novo produto 10x melhor do que qualquer alternativa isolada?"
2. Identifique os dados que você possui que ninguém mais possui — quais produtos adicionais seriam trivialmente melhores se tivessem acesso a esses dados?
3. Para novo produto: forme um time de 2–5 pessoas completamente focadas, sem interrupção das outras áreas. Esse time deve estudar os casos mais complexos primeiro, não o mais simples.
4. Faça o novo PM/founding engineer entrar por 2–3 meses estudando a plataforma e conversando com outros times antes de começar a construir.
5. Defina o design para o caso mais complexo: se global payroll, estude 6 países com sistemas radicalmente diferentes, não apenas os mais parecidos com o seu mercado principal.
6. Meça: "O que podemos fazer agora que era impossível antes desta plataforma?" Isso é o diferencial real — não features.
