# Construir para o Melhor Usuário

**Fluxo:** [[04 - Design & UX]] · **Tema:** [[Design — Experiência & fricção]] · **Camada:** L3
**Tipo:** Mindset · **Fontes:** [[Eeke de Milliano]]

## Ideia central
**Construa para o seu melhor usuário, não para o pior.** É fácil se deixar levar por "e se houver abuso?" ou "e se o usuário não entender?" e moldar o produto de formas estranhas para lidar com edge cases que representam uma fração minúscula dos usuários.

**O problema com construir para o pior usuário:**
- Processos de onboarding inflados tentando converter usuários inadequados para o produto
- Features de proteção/moderação que adicionam fricção para todos por causa de poucos
- Decisões de produto tomadas com base em exceções, não na regra

**A pergunta correta:**
"Não seria incrível ter esse problema?" — Se você está considerando adicionar complexidade para evitar abuso, pergunte se ter esse nível de adoção/abuso não seria um problema excepcional de se ter.

**Especialmente para onboarding:**
Construa o onboarding para o usuário que vai entrar e imediatamente entender o produto. Não tente converter os que não são adequados — otimize para os que são. Na fase inicial de produto, simplesmente não vale a pena se preocupar com isso.

**O momento em que a regra muda:**
Quando você está otimizando funis em escala, faz sentido olhar para a cauda. Mas no desenvolvimento inicial de produto, o foco deve ser no usuário ideal.

## Como aplicar
- Antes de adicionar qualquer feature de proteção/moderação, pergunte: "Quantos % dos meus usuários vão se beneficiar vs. quantos serão impactados negativamente?"
- Para onboarding: "Quem é o usuário que entra e imediatamente faz 'isso é exatamente o que eu precisava'? Construa para ele."
- "Quais decisões de produto estamos tomando com base em 5% dos usuários que atrapalham 95%?"
