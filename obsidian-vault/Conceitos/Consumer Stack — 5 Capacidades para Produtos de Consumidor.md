# Consumer Stack — 5 Capacidades para Produtos de Consumidor

**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Manik Gupta]]

## Ideia central
Não existe playbook garantido para construir produtos de consumidor bem-sucedidos — mas existe um conjunto de capacidades que maximiza as chances. O Consumer Stack é um scorecard de 5 dimensões que qualquer time responsável por produto de consumidor deveria avaliar honestamente.

## As 5 capacidades
1. **Design-led thinking para encantar usuários**: craftsmanship nível A+. Produtos mal desenhados não têm chance. É atenção ao detalhe pixel a pixel, transições entre telas, cada micro-interação.
2. **Foco e priorização rigorosos**: critical user journeys em primeiro lugar. Poucos features, bem feitos. A tendência de "adicionar mais features" destrói a experiência do consumidor.
3. **Métricas e instrumentação corretas**: definir a métrica *e* instrumentá-la de forma inequívoca. Debates sobre "o que é um DAU ativo?" são falha de instrumentação, não de definição.
4. **Alta velocidade de ship e cultura de experimento**: se você não está aprendendo, não está fazendo nada bem. Ciclos curtos de build → ship → medir → iterar.
5. **Talento forte em todas as funções**: PM, design, dados, engenharia, marketing. Cada função precisa ter pessoas que entendem e têm empatia pelo consumidor.

## Como aplicar
- Use como **scorecard trimestral**: dê notas de A a D para cada dimensão com seu time.
- Identifique qual dimensão está abaixo de B e ataque primeiro — ela é o gargalo.
- Aplique mesmo em grandes empresas lançando novos produtos de consumidor.
