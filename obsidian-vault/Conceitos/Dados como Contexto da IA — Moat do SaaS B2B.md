# Dados como Contexto da IA — Moat do SaaS B2B

**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Insight · **Fontes:** [[Shaun Clowes]]

## Ideia central
Dois insights conectados que mudam como se pensa sobre produto e competição na era da IA:

**1. LLMs são tão bons quanto os dados que recebem.** O modelo em si é mais ou menos substituível. O diferencial real está no contexto — toda a informação relevante (dados de clientes, regras do negócio, histórico, legislação local) que você consegue fornecer ao modelo. 90% das calorias em produtos de IA vão para gerenciamento de dados, não para o modelo. Quem entende isso constrói moat; quem ignora fica lutando com qualidade de dados para sempre.

**2. O moat do SaaS B2B nunca foi a UI — são as regras de negócio acumuladas.** Apps B2B são vistos como "forms on databases" e subestimados. Mas o valor real não está na interface: está nos anos de configuração acumulada das regras e workflows específicos de cada cliente. O Salesforce de uma empresa *se torna* aquela empresa — a ponto de ninguém mais entender como funciona sem ler o código. IA não destrói isso; amplifica: os agentes precisam operar *contra as regras do sistema* para saber o que é permitido.

## LLMs como máquinas de síntese — mas vazias sem dados
- Informação tem taxa de decaimento: feedback de cliente de 3 meses é menos útil que de ontem.
- A síntese é onde os LLMs brilham — mas síntese de quê? De dados que você trouxe. "LLMs can only be as good as the data they are given and how recent that data is."
- Você nunca tem dados em excesso para um LLM; quanto mais contexto, melhor o output.
- No produto: não é o modelo que diferencia. São os prompts + todos os dados contextuais (benefícios do funcionário, política da empresa, regulação do país). Quem consegue resolver o problema de dados vence.

## Por que SaaS B2B fica mais forte com IA, não mais fraco
- IA barateia construir "forms on databases" → mais competidores → clientes escolhem o líder estabelecido (ninguém é demitido por comprar Salesforce).
- Agentes podem substituir usuários humanos na UI → mas ainda precisam das regras de negócio para saber o que fazer. Salesforce sem cabeça (sem UI) ainda tem todas as regras — e é exatamente isso que orienta o agente.
- Startups que querem desafiar incumbentes precisam trazer *dados como cidadão de primeira classe*: não apenas UI melhor, mas dados integrados ao workflow (caso Ashby no ATS: dados de outcome de contratação dentro do processo de hiring).

## Dois caminhos prováveis
1. **Incumbentes ficam mais fortes**: quem tem o sistema de record também tem o contexto para treinar agentes melhores.
2. **Nova camada de plataforma**: apps que unem múltiplas funções de negócio com regras + dados integrados — mais horizontal, mas hoje inexistente.

O que *não* vai acontecer: flood de novos SaaS clones baratos ganhando market share dos incumbentes.

## Insights por autor
### [[Shaun Clowes]]
- "LLMs can only be as good as the data they are given and how recent that data is. They're ultimately like information shredders — limitless information eaters."
- "It's a data management problem. Getting access to good data, timely data, well-structured data and getting it to the LLM — that's where 90% of the calories go."
- "Nobody ever gets fired for buying Salesforce. More competitors just give more power to the existing winning systems of record."
- "The value isn't in the UI and it isn't in the data model. It's the years and years of evolution of the underlying workflows — and the customers evolving those workflows to make them work the way they do."
- "If you go to Salesforce and say 'define all the processes by which software was sold inside Salesforce,' they couldn't tell you without reading the code of their Salesforce instance."
- "Agents have to operate against the rules of the system — and the rules are defined by the business processes. They're almost telling the agent what it should do."
