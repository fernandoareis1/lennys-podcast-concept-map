# Data-Driven como Red Flag

**Fluxo:** [[05 - Experimentação & Dados]] · **Tema:** [[Experimentação — Aprender com pouco e qualitativo]] · **Camada:** L3
**Tipo:** insight contrarian · **Fontes:** [[Maggie Crowley]]

## Ideia central
Quando um executivo ou time se define como "muito data-driven", isso pode ser um sinal de alarme: indica sobre-ênfase em dados quantitativos às custas do qualitativo, pouca pesquisa direta com usuários e gestão por dashboard em vez de por compreensão. Dados dizem *o que* está acontecendo; não dizem *por quê*.

## Como aplicar
- Mantenha instrumentação e dashboards — não é anti-dados; é anti-exclusivamente-dados.
- Equilibre: para cada insight quantitativo relevante, conduza entrevistas ou observações que expliquem o *porquê*.
- Use o "obviously better test": se algo é logicamente melhor para o usuário e a lógica é sólida, pode ser desnecessário esperar um A/B test.
- Questione quando alguém bloquear uma decisão com "precisamos de dados" para algo que é visualmente ou logicamente superior.
- Prefira "10 entrevistas com usuários" a qualquer dashboard para entender o comportamento e gerar hipóteses.
