# Day One vs Day 100 Features

**Fluxo:** [[08 - Monetização & Pricing]] · **Tema:** [[Monetização — Estratégia de monetização]] · **Camada:** L3
**Tipo:** framework · **Fontes:** [[Naomi Ionita]]

## Ideia central
Features de **Day One** são as que o usuário consegue apreciar na primeira sessão — elas pertencem ao plano grátis/entrada porque viabilizam o aha moment. Features de **Day 100** são as cujo valor só aparece com escala de uso, dados ou colaboração — elas devem ficar nos planos pagos, onde se tornam incentivo natural de upgrade.

## Como aplicar
1. Liste todas as features do produto e classifique-as: "o usuário consegue ver o valor disso na primeira semana?"
2. Features de Day One que estão no plano pago podem estar bloqueando ativação; considere movê-las para o free.
3. Features de Day 100 no plano grátis estão desperdiçando potencial de upsell; mova-as para planos superiores.
4. Resultado esperado: aumento simultâneo de **upgrade rate** (mais conversão para pago) e **ARPU** (valor por usuário pago) — Invoice2go obteve o dobro da taxa de upgrade com 30% a mais de preço após esse rebalanceamento.
5. Esse framework ajuda a simplificar a pricing page: o free é sobre primeiros passos; o pro/enterprise é sobre poder avançado.

## Insights por autor
### [[Naomi Ionita]]
- Desenvolvido e validado no Invoice2go: dobraram a taxa de upgrade do plano starter para pro enquanto aumentavam o preço do pro em ~30%.
- "Day 100 features têm valor derivado de uma certa escala de dados na plataforma — não faz sentido expô-las no onboarding quando o usuário não vai entender."
- Contrasta com features de Day One (ex.: subir foto de perfil, criar uma fatura básica) que entregam valor imediato e devem remover barreiras de entrada.
