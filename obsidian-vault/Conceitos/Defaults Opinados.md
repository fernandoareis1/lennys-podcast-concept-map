# Defaults Opinados

**Fluxo:** [[07 - Ativação & Retenção]] · **Tema:** [[Retenção — Ativação & onboarding]] · **Camada:** L3
**Tipo:** Técnica · **Fontes:** [[Adam Fishman]]

## Ideia central
Defaults opinados são configurações iniciais do produto baseadas em dados agregados do que funciona para a maioria dos usuários — mas que **não eliminam a escolha**. A ideia: dificultar o caminho errado e facilitar o correto. Isso transfere o conhecimento coletivo do time (aprendido via intervenção humana ou dados) para a experiência do produto de forma escalável.

## Como aplicar
1. Use intervenção humana para descobrir o que funciona (e.g., consultores que ajudam criadores a configurar suas páginas)
2. Identifique os padrões de sucesso (e.g., 3 tiers de preço · preço mínimo $3–$5, não $1)
3. Configure esses padrões como default — com mais fricção se o usuário quiser mudar
4. Torne a opção "errada" disponível, porém mais difícil de acessar (não bloquear, mas desincentiva)

**Regra de ouro:** só redesenhe defaults quando tiver insight net-new sobre usuários — não mude por moda ou por redesign.
