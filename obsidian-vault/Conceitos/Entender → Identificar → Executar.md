# Entender → Identificar → Executar

**Fluxo:** [[06 - Growth & Aquisição]] · **Tema:** [[Growth — Modelos de crescimento]] · **Camada:** L3
**Tipo:** framework · **Fontes:** [[Naomi Gleit]]

## Ideia central
Antes de executar qualquer iniciativa de growth, é preciso **entender** o que os dados dizem. A maioria dos times de growth pula para execução sem instrumentar adequadamente os fluxos — e então não sabe o que de fato move o ponteiro. O framework de três etapas do Facebook growth team de 2009: (1) **Entender** — instrumente tudo; (2) **Identificar** — descubra as maiores oportunidades nos dados; (3) **Executar** — construa os produtos que removem os gargalos encontrados.

## Como aplicar
1. **Entender — pare e instrumente:**
   - Antes de qualquer nova iniciativa, pergunte: "Temos os dados de que precisamos para tomar decisões informadas?"
   - Se não, pause o roadmap e instrumente todos os fluxos relevantes (registro, onboarding, convites, notificações). O Facebook fez exatamente isso em janeiro de 2009 — parou tudo por um mês só para instrumentar.
   - Meça cada passo: onde há drop-off? Qual é a taxa de confirmação de e-mail? Quantos usuários chegam ao "aha moment" em X dias?

2. **Identificar — encontre as maiores oportunidades:**
   - Com os dados em mãos, identifique onde estão as maiores perdas: "20% dos usuários não confirmam o e-mail" é uma oportunidade clara.
   - Use o **growth accounting** como bússola: crescimento líquido = novos usuários − inativos + ressuscitados. As linhas de churn e ressurreição costumam ser maiores que a linha de aquisição — o que indica que retenção é a maior alavanca.

3. **Executar — construa os produtos certos:**
   - Construa funcionalidades que removem as fricções identificadas: NUX (new user experience) para garantir onboarding, importação de contatos para acelerar friending, alternativas à confirmação de e-mail.
   - A métrica de ativação (ex.: 7 amigos em 10 dias) funciona não por ser matematicamente perfeita, mas por criar **clareza extrema** em torno de um objetivo comum.

## Insights por autor
### [[Naomi Gleit]]
- "A pergunta que você deve se fazer é: você tem os dados de que precisa para saber o que fazer em growth? Se não, pare tudo e instrumente."
- "Em 2009 paramos tudo no growth team por um mês só para instrumentar os fluxos. Foi quando descobrimos que 20% dos usuários não confirmavam o e-mail."
