# Escolher o Caminho Difícil — Ganhar Mesmo Perdendo

**Fluxo:** [[10 - Founder & Carreira]] · **Tema:** [[Founder — Carreira & progresso]] · **Camada:** L3
**Tipo:** Insight · **Fontes:** [[Farhan Thawar]]

## Ideia central
Quando você tem múltiplas opções, incline-se pelo caminho mais difícil. A lógica é assimétrica: se o caminho fácil falha, você perdeu o aposto *e* não aprendeu nada. Se o caminho difícil falha, você ainda ganhou — porque trabalhou com pessoas mais inteligentes, desenvolveu skills mais raras e ficou num estado constante de aprendizado.

## A lógica assimétrica

| | Caminho Fácil | Caminho Difícil |
|---|---|---|
| Funciona | ✅ ganhou | ✅ ganhou mais |
| Não funciona | ❌ perdeu e não aprendeu | ✅ aprendeu e cresceu |

O caminho fácil tem retorno assimétrico negativo: o upside é quase o mesmo que o difícil, mas o downside é perder *e* desperdiçar tempo.

## Exemplos de Farhan
- **Job seekers**: em vez de enviar 10 currículos/dia, construa um app na API da empresa que quer entrar. Mesmo sem o emprego, aprendeu, tem código no GitHub e algo para mostrar
- **Na faculdade**: escolheu cursos mais difíceis → notas piores, mas amigos mais inteligentes que ficam para a vida toda
- **MBA em financial engineering**: fez como tech person entre financistas — todos perguntavam "por quê você está aqui?" — mas o networking valeu décadas
- **Carreira**: trabalhou deliberadamente com "unreasonable people" (visão irracional) porque reconheceu que é melhor em execução do que em visão

## Não é apenas "harder"
A distinção importante: difícil pelo caminho certo. Não é se torturar com trabalho dumb ou empresa terrível. É:
- Escolher o problema onde as pessoas mais inteligentes estão
- Escolher o que vai te ensinar algo que você não saberia de outra forma
- Escolher onde o upside do aprendizado é alto mesmo se o objetivo primário falhar
