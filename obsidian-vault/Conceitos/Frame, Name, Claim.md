# Frame, Name, Claim
**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Posicionamento & diferenciação]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Christopher Lochhead]]

## Ideia central
O processo tático central do category design: **Frame** o problema de um jeito novo, **Name** a nova categoria com linguagem que não existia antes, **Claim** o espaço antes que qualquer um perceba que existe.

**Frame — reencadrar o problema:**
O problema que você resolve precisa ser enunciado de uma forma que o atual líder de mercado não consegue resolver. Não é "fazemos X melhor" — é "o problema real é Y, que ninguém percebeu ainda".

Exemplos clássicos:
- **Gojo** (produto pré-reformulado): sabão líquido. Pergunta de reframe: "O problema é que as barras de sabão são difíceis de usar, ou o problema é que as mãos ficam sujas de jeitos que sabão em barra não resolve?"
- **Elisha Otis**: não inventou o elevador, inventou o *elevador de segurança*, depois renomeou para *ferrovia vertical* — o que criou o mercado para arranha-céus. Sem o frame, é apenas um elevador mais seguro
- **Lomi**: não é uma composteira melhor, é um *smart home composter* que transforma lixo orgânico em terra rica em casa — nova categoria, novo problema (lixo fedorento na cozinha + culpa ambiental)

**Name — criar linguagem nova (Languaging):**
Nomear uma categoria não é marketing; é criar scaffolding mental que permite ao mercado pensar sobre o problema de uma nova maneira. Uma vez que a linguagem existe, outros a adotam — e quem cunhou a linguagem lidera.

- **Starbucks** criou terminologia italiana (*venti*, *barista*) para fazer o americano pagar $3 num café de $0.25 — a linguagem criou a experiência premium antes do produto
- **OpenAI** criou "LLM", "training data", "hallucination", "prompt engineering" — hoje toda conversa sobre IA usa vocabulário OpenAI, o que mantém a empresa no centro mental da categoria
- Linguagem não descreve realidade; *cria* realidade no mercado

**Claim — plantar a bandeira:**
Depois de reencadrar e nomear, você precisa declarar explicitamente que você é o dono da categoria. Não é arrogância — é necessário para que o mercado organize sua cognição em torno de você como referência.

## Como aplicar
1. **Escreva o Frame em uma frase:** "O problema real não é [problema antigo], é [novo problema que você define]"
2. **Invente pelo menos 3 termos novos** para descrever a categoria; teste quais ressoam com seus super consumers; adote o que grudar
3. **Crie um "Category Point of View" (POV):** documento que articula o novo problema, por que as soluções existentes são inadequadas, e por que sua solução define a nova categoria — publicado, não interno
4. **Teste o Frame com clientes:** apresente o problema sem mencionar sua solução. Se eles disserem "isso é exatamente o meu problema, mas eu nunca pensei assim" — você acertou o frame
5. **Claim via eventos:** use lightning strikes (lançamentos concentrados) para plantar a bandeira publicamente
