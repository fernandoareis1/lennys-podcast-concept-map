# Freemium reverso
**Fluxo:** [[08 - Monetização & Pricing]] · **Tema:** [[Monetização — Estratégia de monetização]] · **Camada:** L3
**Tipo:** Técnica · **Fontes:** [[Albert Cheng]]

## Ideia central
Em produtos freemium, o erro clássico é que o plano gratuito apresenta apenas features básicas — o que limita a percepção de valor do produto e torna a conversão difícil. O **freemium reverso** inverte isso: mostra ao usuário gratuito uma *amostra* das features premium em tempo real, no contexto de uso. Em vez de prender o usuário num produto degradado, você o deixa experimentar o produto completo em doses limitadas. Resultado: o usuário entende o que está comprando antes de pagar, e a conversão sobe dramaticamente.

## Como aplicar
1. Mapeie quais features premium geram o maior "aha" para o usuário — o que faz ele dizer "isso sim é diferente"?
2. Intersperse amostras dessas features para usuários free, no contexto de uso real (não em tour separado).
3. Limite a quantidade (caps) mas não a qualidade — o usuário deve ver o produto completo, não uma versão degradada.
4. Instrumente bem: rastreie quais sugestões/features os usuários free estão vendo. Sem dados não há otimização.
5. Validar: se o produto free já reflete o que o produto premium oferece, você não está "dando demais" — está educando o usuário e acelerando a decisão de compra.

## Insights por autor
### [[Albert Cheng]]
- Grammarly: usuários free viam o produto como "ferramenta de gramática e ortografia" porque as sugestões free eram só correções básicas. A percepção não refletia a IA de escrita avançada que a empresa havia construído.
- Mudança: intersperse sugestões premium (reescrever frases, tom, clareza) para usuários free em seus textos reais. Resultado: taxas de upgrade *quase dobraram*.
- "Certifique-se de que o produto gratuito seja um reflexo de tudo que o produto pode oferecer — geralmente valerá a pena."
- É como um "trial reverso em tempo real": em vez de uma janela de tempo, o usuário recebe uma dose de valor premium a cada sessão de escrita.
- Premissa de negócio para freemium: missão ampla (Duolingo, Grammarly, Chess.com) requer escala máxima → produto gratuito é essencial; monetização premium é a camada de cima.
