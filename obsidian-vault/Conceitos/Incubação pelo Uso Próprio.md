# Incubação pelo Uso Próprio
**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Dan Shipper]]

## Ideia central
O modelo de incubação de produto da Every: **descubra o produto vivendo no edge da IA, não na pesquisa de mercado**.

O processo em três passos:
1. **Descoberta pelo uso:** enquanto opera uma empresa AI-first, você topa com necessidades que não existiam antes — "eu precisava de um ghost writer mas é caro; então usei o Claude para isso." Essa fricção é o sinal de produto
2. **Validação interna:** use as ferramentas genéricas (ChatGPT, Claude) para resolver a necessidade. Se todo o time de 15 pessoas adota espontaneamente → "é um banger internamente" → sinal forte
3. **Unbundle para produto:** extraia a necessidade de dentro do ChatGPT e construa um produto focado para ela. Quem lê a Every tem o mesmo perfil que a equipe → eles se tornam os próximos usuários

**Por que agora o game board zerou:**
- Há 5 anos: qualquer produto novo era "mais um app de notas" ou "mais um SaaS B2B"
- Agora: o territory é completamente novo, ninguém sabe o que funciona, workflows estão sendo inventados em tempo real
- Analogia: quando planilhas surgiram, levou anos para descobrirmos todos os novos workflows possíveis; o mesmo está acontecendo com IA

**GPT wrappers são negócios legítimos:**
Dan é categórico: wrappers de GPT têm sido difamados sem motivo. A crítica ignora que:
- Identificar a necessidade específica vale muito
- A interface e o contexto certo fazem toda a diferença
- O histórico mostra que "unbundling" de ferramentas gerais gera negócios enormes (o B2B SaaS foi unbundling de Excel e Access)

**Software está se tornando conteúdo:**
Haverá formas de software que não parecem software hoje mas podem ser negócios — skills de browser (DIA), custom GPTs, apps no-code que um não-técnico pode criar e monetizar. A diferença entre Hollywood e YouTube: ambos são vídeo, mas escalas e modelos de negócio completamente diferentes.

**O modelo de funding correto:**
Para uma empresa que incuba, VC tradicional e "coloque 50M no banco" é errado — o dinheiro empurra para escalar antes de saber o que você tem. A Every usou "sip seed": comprometimento de até 2M que pode ser sacado quando precisar, sem pressão de burn.

## Como aplicar
1. **Opere na edge:** a única forma de descobrir produtos no-edge de IA é usar IA para o seu próprio trabalho real, todo dia. A descoberta é um subproduto da operação, não de pesquisa
2. **Documente as fricções:** quando você usa uma ferramenta geral para resolver uma necessidade específica repetidamente, anote: "preciso do ChatGPT para X toda semana" — isso é sinal de produto
3. **Teste com seu próprio time primeiro:** se todos na empresa adotam espontaneamente (sem você precisar pedir), é sinal forte o suficiente para a próxima fase
4. **Não descarte wrappers:** a pergunta não é "isso é apenas um wrapper de GPT?" — é "qual é o nível de dor que resolve e existe demanda real?" 
5. **Considere o sip seed:** se você está incubando múltiplas coisas e não quer pressão de VC, uma linha de crédito que você saca conforme precisa pode ser melhor que capital upfront
