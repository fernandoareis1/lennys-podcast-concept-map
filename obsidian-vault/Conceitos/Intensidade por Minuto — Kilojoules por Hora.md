# Intensidade por Minuto — Kilojoules por Hora

**Fluxo:** [[09 - Liderança & Times]] · **Tema:** [[Liderança — Modelos de time]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Farhan Thawar]]

## Ideia central
A maior alavanca de produtividade não é trabalhar mais horas — é **gastar mais energia por hora**. Uma hora é uma hora para qualquer pessoa. A diferença entre um time bom e um ótimo está na densidade de trabalho real dentro de cada hora, não na quantidade de horas. "What if you just did more per minute?"

## Lei de Parkinson aplicada
O trabalho se expande para preencher o tempo disponível. Use isso a favor:
- Shrink timelines → o time comprime o trabalho para caber
- Weekly GSD updates → toda semana há algo a mostrar; ninguém quer aparecer sem progresso
- Six-week reviews com Toby → frequência suficiente para criar urgência, suficientemente longa para resultados reais

## Práticas da Shopify para intensidade
1. **GSD (Get Shit Done)** — updates semanais de cada projeto para toda a empresa; cria ciclo de accountability semanal
2. **Pair programming** — dois pares em foco profundo > um par em foco intermitente durante dobro do tempo
3. **Six-week reviews** — ao invés de QBRs/semestrais; cada time apresenta roadmap + resourcing + progresso para liderança e Toby
4. **No change management** — ao invés de gerir mudanças por 6 meses, implementam na semana; "it's going to leak anyway, just launch it"
5. **Delete Code Club** — sempre encontram mais de 1 milhão de linhas para deletar; código é uma liability, não um ativo

## Por que não basta trabalhar mais
- Trabalhar 12h/dia com foosball no meio = mesma energia que 8h focadas
- Pessoas excelentes entregam alta qualidade em menos tempo: "there is a time variance that people don't think about"
- Parkinson's Law favorece: se você der 1h para uma tarefa, levará 1h; se der 4h, levará 4h
