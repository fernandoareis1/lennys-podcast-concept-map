# Intentionalidade Primeiro — O GPS não Escolhe o Destino

**Fluxo:** [[05 - Experimentação & Dados]] · **Tema:** [[Experimentação — AB testing & cultura]] · **Camada:** L3
**Tipo:** Mindset · **Fontes:** [[Sanchan Saxena]]

## Ideia central
A/B tests não servem para decidir para onde ir — servem para encontrar o caminho mais rápido até o destino que você já escolheu. A analogia: quando você entra no carro, você diz ao GPS "quero ir a São Francisco" e o GPS encontra a melhor rota. Você nunca pergunta ao GPS "para onde eu deveria ir?" O mesmo vale para produto: o A/B test é a rota. A **intentionalidade** — a visão clara de que experiência você quer criar para o cliente — é o destino.

**A confusão comum:** PMs que usam A/B tests para validar se deveriam tentar algo estão colocando a ferramenta no lugar errado. Isso cria dois problemas: (1) mata ideias cedo porque "os dados não mostraram movimento"; (2) faz o time otimizar localmente sem saber onde está indo.

**O caso Instagram Stories:** Kevin Systrom queria criar "o maior canal de TV em tempo real do mundo". Data science apresentou dados dizendo que 99% das fotos no celular são de uma semana atrás — as pessoas querem compartilhar memórias antigas. Usuários disseram o mesmo. Systrom disse não: "a intentionalidade é que stories é o que está acontecendo AGORA." Resultado: Instagram Stories proibiu fotos com mais de 24h. Depois que ele saiu, isso foi removido — uma intentionalidade diferente.

**A sequência certa:** (1) Defina a intentionalidade — qual mundo você quer criar para o seu cliente? (2) Use A/B tests para validar que você está indo na direção certa. (3) Itere. Mas não abandone a intentionalidade cedo demais: ideias brilhantes morrem por falta de paciência de ver o resultado.

**Intuição como dado pré-estatístico:** gut e intuição são dados — só não são estatisticamente significativos ainda. Você construiu essa intuição porque se imergiu no problema, conversou com clientes, viveu o contexto. Não desconte isso só porque não tem p-value.

## Como aplicar
1. **Antes de qualquer A/B test, escreva a intentionalidade**: "a experiência que quero criar para o cliente é ____." Teste valida a rota, não o destino.
2. **Distingua "testando se vale a pena" de "encontrando o melhor caminho"**: o primeiro é problema de strategy/discovery; o segundo é onde o A/B test é certo.
3. **Dê tempo para ideias brilhantes amadurecerem**: "impaciência para começar, paciência para ver o resultado." Matar ideias cedo por dados prematuros é um dos erros mais caros.
4. **Articule o que sua intuição diz antes de pedir dados**: "sinto que X porque passei tempo com Y usuários e vi Z comportamento." Isso transforma gut em hipótese testável.
5. **Aceite que certas decisões de intentionalidade não cabem em A/B test**: a decisão de colocar Stories no topo do feed da Instagram não foi um A/B test de feature — foi uma aposta de direção de produto.
