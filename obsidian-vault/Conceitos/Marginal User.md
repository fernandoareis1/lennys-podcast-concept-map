# Marginal User
**Fluxo:** [[01 - Discovery & Pesquisa]] · **Tema:** [[Discovery — Descoberta contínua & validação]] · **Camada:** L3
**Tipo:** Técnica · **Fontes:** [[Adriel Frederick]]

## Ideia central
O "marginal user" é a pessoa que está na beira de converter/ativar — tráfego alto, conversão baixa. Estudar o pior caso (o usuário com mais fricção) revela tudo que está errado no produto; priorizar pelo usuário "marginal" (remove as variáveis mais difíceis) revela o que atacar primeiro. Dados apontam onde; observar a pessoa revela o porquê.

## Como aplicar
1. Encontre no funil o segmento com mais tráfego e pior conversão (ex.: país com alta demanda mas baixa taxa de cadastro).
2. Assista esse usuário — não analise os dados dele. Observe alguém navegar ao vivo ou assista sessões gravadas.
3. Liste tudo que está errado (latência, idioma, formato de telefone, fluxo de onboarding).
4. Para priorizar: remova as barreiras que você sabe que não pode resolver (ex.: telefone antigo + conexão edge) e veja quais problemas ainda persistem para a próxima camada acima do pior caso.
5. Dado e observação são complementares — o dado diz "quão ruim"; a observação diz "por quê".

- No Facebook: encontrou que usuários em países distantes de data centers com feature phones eram o pior caso. Revelou: idioma errado, latência, formato de telefone. Cada problema resolvido beneficiou milhões de usuários mais bem equipados.
