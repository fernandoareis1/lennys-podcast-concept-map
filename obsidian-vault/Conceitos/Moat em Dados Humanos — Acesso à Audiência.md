# Moat em Dados Humanos — Acesso à Audiência

**Fluxo:** [[06 - Growth & Aquisição]] · **Tema:** [[Growth — Modelos de crescimento]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Garrett Lord]]

## Ideia central
No mercado de dados humanos para IA, **o único moat real é o acesso a uma audiência**. Não é tecnologia de plataforma, não é modelo proprietário de avaliação de qualidade — é distribuição. Quem já tem uma rede de especialistas confiando na marca não precisa comprar atenção. Todo mundo sem essa rede está gastando dezenas de milhões de dólares por mês tentando replicar o que um network estabelecido tem de graça.

## Estrutura do moat
| Dimensão | Concorrentes sem audiência | Handshake |
|---|---|---|
| Aquisição | 200 recrutadores + performance ads | Parceria com 1.600 universidades |
| CAC | Dezenas de M$/mês | ~Zero |
| Conversão | Baixa (sem brand/trust) | Alta (marca conhecida dos estudantes) |
| LTV | Baixo (churn alto) | Alto (experiência de qualidade = retenção) |
| Qualidade | Variável | Controlada via treinamento + cohort |

## Por que audiência é difícil de comprar
- Especialistas (PhDs, médicos, advogados) não respondem a spam de LinkedIn
- Eles percebem quando são tratados como mão-de-obra commodity: "The frontier tax accountants don't want to be treated like low-cost international labor"
- Confiança de marca leva anos para construir; marketplaces acadêmicos exigem aprovação institucional (universidades não toleram parceiros que tratam mal seus estudantes)

## Modelo de flywheel
1. Parceria com universidade → acesso a estudantes e pós-graduandos
2. Experiência de qualidade → retenção + recomendação orgânica
3. Dados melhores → mais contratos com frontier labs
4. Mais projetos → mais oportunidade para os especialistas → mais supply
5. Mais supply → maior confiança dos labs → mais demanda

## Insights por autor

- LTV = retenção × número de projetos; tratar bem = LTV alto; não há trade-off entre experiência do trabalhador e margem
