# Momentum Manufaturado — Deadlines Artificiais como Combustível

**Fluxo:** [[09 - Liderança & Times]] · **Tema:** [[Liderança — Cultura & valores]] · **Camada:** L3
**Tipo:** Conceito · **Fontes:** [[Laura Modi]]

## Ideia central
O trabalho do líder não é apenas **manter** momentum — é **criar** momentum. E às vezes o momentum tem que ser manufaturado: deadlines e datas de lançamento declaradas sem motivo externo, apenas para forçar o time a convergir e executar. "Why are we launching this May 1st?" "Just because we said it. Because if we don't do it now, we may never do it." A deadline arbitrária cria urgência real, que gera progresso real, que cria o momentum necessário para o próximo passo.

**Por que momentum precisa ser manufaturado:**
- Sem deadlines externas (investidores, contratos, regulação), a tendência é procrastinar indefinidamente
- Perfeccionismo e discussão intermináveis são a morte silenciosa de startups
- "Perfect is the enemy of good" — um deadline força a versão 0.7 a sair, o que revela aprendizados que a versão 1.0 hipotética nunca revelaria
- Momentum tem inércia: times parados ficam parados; times em movimento ficam em movimento

**Como manufaturar momentum:**
- Declare uma data publicamente: comunique a data para o time e para stakeholders — o comprometimento público aumenta o custo de falhar
- Dê o nome certo: "Slowth Team" em vez de "equipe parada"; "Launch de Maio" em vez de "quando estivermos prontos"
- Celebre o momentum, não apenas o resultado: reconheça progresso intermediário para manter energia
- Use milestones artificiais como pontos de revisão: 30-60-90 dias com outputs esperados

**A clareza sobre o que o momentum manufaturado NÃO é:**
- Não é criar pressão insustentável (burnout ≠ momentum)
- Não é mentir para o time sobre por que a data existe
- Não é ignorar aprendizados — o ponto é lançar para aprender, não para acabar
- Ser transparente que a deadline é arbitrária é honesto e funciona: o time entende que o prazo é um mecanismo, não uma crise

## Como aplicar
1. Para qualquer projeto que está em discussão há mais de 2 sprints sem output: declare uma data de "versão 0" — não a versão final, mas algo para mostrar
2. Torne a deadline pública antes de ter certeza que é alcançável — o comprometimento público muda a dinâmica internamente
3. Nomeie o esforço: "Sprint de Fevereiro", "Lançamento Mínimo de Março" — o nome cria identidade e urgência
4. Quando o time perguntar "por que essa data?", seja honesto: "Porque se não fizermos agora, não vamos fazer nunca. A data nos obriga a converger."
5. Após cada lançamento manufaturado, celebrate: documente o que foi aprendido e use para justificar o próximo ciclo de momentum
