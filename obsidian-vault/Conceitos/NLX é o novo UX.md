# NLX é o novo UX
**Fluxo:** [[04 - Design & UX]] · **Tema:** [[Design — Experiência & fricção]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Aparna Chennapragada]]

## Ideia central
**NLX (Natural Language Experience) é a nova fronteira do design de interface.** Interfaces gráficas (GUI) foram sempre explicitamente projetadas porque eram não-naturais — menus, dropdowns, grids são construtos artificiais visíveis. Interfaces conversacionais em linguagem natural parecem orgânicas, mas têm gramáticas, estruturas e "elementos de UI" igualmente ricos — só que invisíveis. Errar ao não projetá-los conscientemente é criar experiências caóticas ou "Frankenstein." O design de NLX inclui: **prompt** como novo elemento de UI; **plano editável** quando o agente recebe objetivo de alto nível; **show the work** (quanto mostrar do raciocínio sem sobrecarregar); **follow-ups proativos** no momento certo; e **ritmo e tom** da conversa.

## Como aplicar
1. **Trate o prompt como elemento de UX**: qual é o prompt ideal para este fluxo? Quais instruções predefinidas ou sugestões ajudam usuários a chegar lá?
2. **Projete "mostrar o trabalho" deliberadamente**: muito verbose → parece cron job assustador; muito silencioso → perda de confiança. Calibre a quantidade de "pensamento visível" para a fase de adoção e perfil do usuário.
3. **Defina follow-ups como UI**: quando o agente entrega um resultado, que 2-3 follow-ups naturais deve sugerir? Excesso = irritante; escassez = oportunidade perdida de guiar para um happy path.
4. **Projete planos editáveis**: para tarefas complexas de agente, mostrar um plano antes de executar (e permitir edição) é um novo padrão de UI — como um "modal de confirmação" evoluído.
5. **Personalize a verbosidade por perfil**: a interface ideal de NLX varia entre usuários — uns querem o TLDR, outros querem ver o raciocínio completo. Projete para ambos.

## Insights por autor
### [[Aparna Chennapragada]]
- "Conversas também têm gramáticas, estruturas, elementos de UI — são invisíveis. O que são os novos princípios, os novos construtos da linguagem natural como interface?"
- Exemplos de construtos NLX já emergindo: prompts (novo elemento como dropdown ou menu foi), planos editáveis, thinking aloud (mostrar raciocínio), follow-ups sugeridos.
- "Se um produto de AI fosse apenas 'você chata com ele', significa que não foi projetado. Não é verdade. Tem design profundo nessa conversa."
- Ponto de personalization: "Meu 'API' seria muito diferente do de outros. Alguns querem direto ao ponto; outros querem ver o passo-a-passo."
- Paralelo com GUI: assim como interfaces gráficas foram não-intuitivas no início e exigiram décadas de convenções de UX, NLX exigirá seus próprios padrões emergentes.
