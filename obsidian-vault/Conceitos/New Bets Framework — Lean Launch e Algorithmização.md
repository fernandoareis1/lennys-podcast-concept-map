# New Bets Framework — Lean Launch e Algorithmização

**Fluxo:** [[03 - Priorização & Roadmap]] · **Tema:** [[Priorização — Priorização & foco]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Dmitry Zlokazov]]

## Ideia central
O framework de "new bets" da Revolut define como lançar novos produtos dentro de uma empresa grande com velocidade de startup. Qualquer pessoa pode propor uma aposta — não apenas product owners. O processo:

1. **Pitch**: mostre mercado, caso de negócio, problema do cliente, diferencial competitivo da Revolut
2. **Green light rápido**: estrutura flat + acesso a founders = aprovação sem burocracia
3. **Time lean**: poucos pessoas, first version rapidamente para obter feedback real
4. **Polish antes de scale**: nunca scale um produto até que métricas (retenção, engajamento) sejam sólidas
5. **Algorithmização**: formalize o processo em receita replicável para escalar aos 50 países

**Vantagem estrutural da Revolut:**
Novos produtos podem ser multiplicados instantaneamente por 50+ milhões de usuários quando sólidos — não há Cold Start Problem da forma que startups enfrentam.

**Princípio-chave:** Reduza funcionalidades, nunca qualidade. Mesmo a primeira versão deve ser "wow".

## Como aplicar
1. Abra o processo de ideação para toda a empresa — as melhores apostas podem vir de qualquer lugar
2. Mantenha o pitch simples: mercado + caso de negócio + problema do cliente + vantagem competitiva
3. Monte time mínimo para first version — 2-3 pessoas, não um squad completo
4. Defina threshold mínimo de métricas antes de scale (retenção, satisfação)
5. Após validação, invista em "algorithmização": documente o que funcionou e replique sistematicamente
