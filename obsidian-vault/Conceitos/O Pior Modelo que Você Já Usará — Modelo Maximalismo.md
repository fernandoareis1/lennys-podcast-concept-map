# O Pior Modelo que Você Já Usará — Modelo Maximalismo

**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Mindset · **Fontes:** [[Kevin Weil]]

## Ideia central
"O modelo de IA que você está usando hoje é o pior modelo de IA que você vai usar pelo resto da sua vida." Esta frase resume a filosofia de **modelo maximalismo**: construa para as capacidades que estão quase lá, não para as limitações atuais.

A trajetória de melhoria dos modelos é exponencial: GPT-3.5 custava 100x o que custa o GPT-4o mini hoje, para inteligência inferior. Modelos ficam mais inteligentes, mais rápidos, mais baratos e mais seguros a cada iteração — com alucinações reduzindo a cada ciclo. Os modelos da série o (reasoning) lançam a cada 3-4 meses, cada um com salto de capacidade.

**A implicação prática para PMs e founders:**
Não construa andaimes em torno das limitações atuais do modelo. Se um produto "apenas mal funciona" na borda das capacidades do modelo hoje, continue. O modelo vai alcançar você. Construa para onde o puck vai estar, não onde está agora.

**Analogia com transistores:** A Lei de Moore dobrava transistores a cada 18 meses. Os modelos de IA estão melhorando 10x ao ano. É uma exponencial mais íngreme ainda.

## Como aplicar
1. Quando um produto de IA falha 30-40% do tempo, não abandone nem adicione filtros manuais. Avalie se o problema é fundamentalmente o approach ou só a capacidade atual do modelo.
2. Projete a experiência para o modelo de 6-12 meses no futuro (ver [[Construir para o disco futuro]]). Se funciona 20% hoje, vai funcionar 100% em breve.
3. Não otimize custos de API prematuramente: os preços caem em ordens de magnitude a cada 2 anos.
4. Ao apresentar um produto que "ainda não está pronto", use a linguagem do modelo maximalismo: "isso é o pior que vai ser."
5. Evite a armadilha de reconstruir o produto quando o modelo melhorar — desenhe a arquitetura já assumindo a capacidade futura.
