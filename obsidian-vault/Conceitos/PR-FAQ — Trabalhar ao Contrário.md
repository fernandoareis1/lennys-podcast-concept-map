# PR/FAQ — Trabalhar ao Contrário
**Fluxo:** [[03 - Priorização & Roadmap]] · **Tema:** [[Priorização — Estratégia vs execução]] · **Camada:** L3
**Tipo:** Técnica · **Fontes:** [[Bill Carr]]

## Ideia central
Antes de qualquer trabalho de construção, escreve-se um **Press Release + FAQ** descrevendo o produto como se ele já existisse e fosse anunciado ao mundo. O PR tem três parágrafos essenciais: (1) quem é o cliente, (2) qual é o problema deles, (3) qual é a solução. O FAQ responde às perguntas difíceis que surgirão inevitavelmente — técnicas, de negócio, de stakeholders.

A função não é criar material de marketing: é forçar clareza de pensamento **antes** de se tornar prisioneiro do ciclo de desenvolvimento. Se você não consegue escrever um PR convincente, é sinal de que o problema não está claro o suficiente para justificar construção. O processo funciona como funil (muitos PRFAQs entram, poucos saem), não como túnel (onde tudo o que entra é construído).

A revisão em círculos concêntricos (de uma pessoa → para o time → para liderança) serve a dois propósitos: (1) refinamento iterativo da ideia, e (2) alinhamento antes da execução, eliminando surpresas tardias.

## Como aplicar
1. **Escreva o PR com os três parágrafos essenciais**: cliente (específico, não genérico), problema (com dados ou insights de pesquisa), solução (o que o produto faz, não como ele funciona internamente).
2. **O titular é teste de clareza**: se o headline precisa de mais de uma linha para ser entendido, o produto não está claro o suficiente.
3. **Date o PR**: a data imaginária de lançamento sinaliza complexidade percebida — próximo mês = simples; 18 meses = ambicioso. Torna as expectativas explícitas.
4. **Use FAQs como checklist de problemas conhecidos**: cada FAQ que você antecipa é um risco que você pode mitigar antes, não depois.
5. **Revise em círculos concêntricos**: 1 pessoa → equipe pequena → liderança; a ideia se refina ou morre antes de consumir engenharia.
6. **Crie um portfólio de PRFAQs e priorize**: trate como VC trataria startups — não financie tudo, escolha o melhor 10-20%.
