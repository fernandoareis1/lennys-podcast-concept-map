# PSHE — Avaliar Talento além do Escopo

**Fluxo:** [[09 - Liderança & Times]] · **Tema:** [[Liderança — Modelos de time]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Shishir Mehrotra]]

## Ideia central
O erro mais comum em calibração de talento é usar **escopo** como proxy de senioridade. O problema: escopo é um input que o manager controla, não um output do talento. PSHE é a alternativa: avalie o talento por *onde* na cadeia de decisão a pessoa opera — quanto ela define vs. quanto recebe pronto.

**PSHE** = Problem → Solution → How → Execution

- **Execution (E)**: recebe problema + solução + plano de execução definidos. Sua contribuição é executar o playbook. Nível inicial.
- **How (H)**: recebe problema + solução rascunhada. Define como chegar lá — milestones, rituais, comunicação. Nível intermediário.
- **Solution (S)**: recebe o problema. Volta com as soluções. É julgado pela criatividade e eficácia das alternativas. Sênior.
- **Problem (P)**: define os problemas. "Você me disse para trabalhar em ativação, mas acho que nosso problema é qualidade." Topo da progressão; típico de empresários/CEOs.

## A curva de senioridade
Não é linear. Nos primeiros anos de carreira, o crescimento é *horizontal* (mais escopo no mesmo nível E). No meio, a curva vira e a diferença entre L3 e L7 não é tamanho do projeto — é *como* a pessoa opera (de E para P). É a fase mais confusa para o funcionário (tudo que a educação ensinou foi scope-based) e para o calibrador (dois níveis podem fazer "a mesma coisa" mas com PSHEs completamente diferentes). Shishir chama essa fase de **"trough of dissolution"**.

## Aplicação universal
O framework não é específico para PM — funciona para engenheiros, designers, vendedores:
- **Vendedor E**: segue script + território dado.
- **Vendedor H**: define como vender dentro do produto/região.
- **Vendedor S**: propõe qual produto, qual segmento, qual abordagem.
- **Vendedor P**: "estamos abordando o cliente errado" — identifica o problema estratégico antes do gerente.

A correlação: pessoas no nível P são frequentemente as melhores em Eigenquestions — sabem distinguir o que é decisivo do que é apenas curioso.

## Nos reference checks
Mais do que entrevistas, o PSHE se revela nas referências:
- "Quando você precisa de alguém para resolver um problema difícil, você se vira para quem?" → P/S nível
- "Quem é mais eficiente a operacionalizar a execução?" → H/E nível
- Candidatos H-level costumam ter referências que elogiam eficiência de reuniões e comunicação — bom sinal de H, não de P.

## Insights por autor
### [[Shishir Mehrotra]]
- Criado em 2011 para calibrar PMs no Google após transição de organização funcional para BUs — cada BU estava inventando suas próprias definições.
- Desafio que levou ao framework: guide de níveis existente tinha ~30 linhas por nível com critérios arbitrários; ninguém conseguia identificar o nível de um guide cego.
- "The difference between a level three and a level seven may not be scope — they may do the exact same job. It's how they do the job that matters."
- Usado para leveling, entrevistas E reference checks na Coda.
