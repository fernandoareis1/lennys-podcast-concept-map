# Perguntas de Onboarding como Ativação de Benefício

**Fluxo:** [[07 - Ativação & Retenção]] · **Tema:** [[Retenção — Ativação & onboarding]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Kristen Berman]]

## Ideia central
Usuários em onboarding não absorvem benefícios passivamente (carrosséis, telas de boas-vindas, listas de features). **Fazer perguntas durante o signup — perguntas sobre objetivos, preferências e contexto do usuário — força o engajamento ativo com o próprio benefício do produto**, gerando intenção, personalização percebida e comprometimento. A pergunta não é para coletar dado; é para fazer o usuário se comprometer com o uso.

**Por que perguntas funcionam melhor que apresentações:**
- Presente bias: carrossel promete benefício futuro. Pergunta ativa o benefício agora (o usuário *está fazendo* algo útil ao responder)
- Commitment and consistency (Cialdini): ao declarar um objetivo, o usuário se torna mais propenso a agir consistentemente com essa declaração
- Personalização percebida: usuário sente que o produto foi feito para ele
- Cognitive fluency: perguntas fáceis (múltipla escolha, dropdown) pedem esforço baixo mas criam engajamento alto

**Evidências:**
- **Apartment List:** quizzes de preferência de apartamento durante signup → usuários muito mais engajados nas buscas subsequentes
- **Stitch Fix / Trunk Club:** perguntas de estilo no onboarding → **+133% de conversão** nos usuários que completaram o quiz vs. os que pularam
- **TytoCare (telemedicina pediátrica):** quiz sobre saúde do filho → **53% de taxa de compra** para completers vs. **37%** para não-completers
- **One Medical:** perguntas de saúde → recomendação de 1 médico + horários limitados → **+20% de agendamentos** durante o onboarding

**Princípios de design das perguntas:**
1. **Fáceis primeiro:** comece com múltipla escolha ou dropdown, nunca campo de texto aberto
2. **Relevantes para o objetivo do usuário, não da empresa:** "O que você quer alcançar?" vs. "Para qual use case você vai usar?"
3. **Mostre que a resposta importa:** use a resposta para personalizar o próximo passo imediatamente
4. **Não pergunte demais:** 3-5 perguntas é o limite antes do drop-off aumentar significativamente
5. **Ofereça skip, mas sinalize o custo:** "Pule (você terá uma experiência genérica)"

## Como aplicar
1. Identifique o objetivo do usuário no momento de signup — o que ele está esperando conquistar?
2. Desenhe 3-5 perguntas que o façam declarar esse objetivo ou preferências relacionadas (múltipla escolha, dropdown)
3. Use cada resposta para personalizar o primeiro passo no produto (recomende 1 item, não 10)
4. Teste uma versão com perguntas vs. uma versão com carrossel informativo — meça ativação 7-day e 30-day
5. Monitore completion rate das perguntas. Drop-off em uma pergunta específica = pergunta muito difícil ou não relevante
6. Revise perguntas a cada quarter — os objetivos dos usuários evoluem com o produto

## Insights por autor
### [[Kristen Berman]]
- "Carrossel de onboarding é uma das maiores perdas de tempo em produto. Ninguém lê. Ninguém se lembra. Ninguém age."
- A lógica contra-intuitiva: adicionar fricção (perguntas) no início do onboarding pode aumentar a ativação porque o usuário está investindo ativamente
- One Medical foi um caso especial: a pergunta de saúde + a recomendação de um único médico (não uma lista) + horários limitados criaram um cocktail de comprometimento e urgência
- "Perguntas fazem o usuário sentir que o produto está sendo construído para ele naquele momento — isso é onboarding mágico"
- A regra de ouro de Kristen: as perguntas devem ser sobre o usuário, não sobre como o produto vai te ajudar. "O que você quer alcançar?" não "Sabia que temos X feature?"
