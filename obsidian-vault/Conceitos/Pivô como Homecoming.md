# Pivô como Homecoming
**Fluxo:** [[10 - Founder & Carreira]] · **Tema:** [[Founder — Mentalidade de founder]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Dalton Caldwell]]

## Ideia central
Um bom pivô não é uma fuga do que está falhando — é um retorno ao que você genuinamente conhece. "A good pivot is like going home. It's warmer, it's closer to something that you're an expert at."

**As duas condições de um bom pivô:**
1. **Mais quente (warmer):** o novo problema está mais próximo de onde você tem expertise real, não mais longe
2. **Constrói sobre o que você aprendeu:** as últimas N meses de trabalho no problema anterior geraram algum insight ou expertise que você leva para a nova ideia

**Os casos exemplares:**
- **Brex:** co-fundadores tinham construído uma fintech no Brasil quando mais jovens → pivot de VR headset para cartão corporativo. O conhecimento de pagamentos e compliance que eles achavam irrelevante virou o edge
- **Retool:** tinham construído dashboards internos para operar o Cashew (clone de Venmo) → pivot para ferramentas internas. Fizeram para si mesmos o produto que depois venderam
- **Segment:** começaram com software para faculdades → pivot para concorrente do Mixpanel (usando o que aprenderam sobre analytics) → pivot para SDK de eventos (porque ninguém queria o Mixpanel deles, mas queriam o JavaScript que mandava eventos para múltiplos destinos). Cada pivot construiu sobre o anterior

**O insight sobre o Segment é crucial:**
Não havia universo onde os fundadores do Segment teriam começado com a ideia final. Eles não sabiam nada sobre como analytics funciona antes de tentar vender para faculdades. O conhecimento que gerou a ideia final foi um side effect de tentar e falhar nas ideias anteriores.

**Quando é hora de pivotar:**
- Você está sem ideias de como fazer a empresa atual crescer (o sinal mais claro)
- Você tem expertise subutilizada que poderia ser o alicerce de um problema diferente
- Você identifica um problema que já conhece de perto, mas que conscientemente "excluiu" como ideia de startup

**Como identificar o pivô certo:**
Procure o problema que você inconscientemente descartou — seja porque estava "muito próximo" ou "muito óbvio" ou porque você estava "queimado" de trabalhar naquilo. Frequentemente a resistência interna é sinal de que é exatamente onde você tem o edge que os outros não têm.

## Como aplicar
1. **Mapeie o que você aprendeu nos últimos 6-12 meses de trabalho:** qual expertise foi construída? Quais insights são únicos seus?
2. **Identifique problemas que você descartou:** o que você sabe muito bem que "nunca pensou" como startup idea por ser óbvio demais?
3. **Teste de temperatura:** a ideia do pivô está mais quente (mais próxima do seu conhecimento) ou mais fria (indo para território desconhecido)?
4. **Use a Zip strategy:** se a direção ainda não está clara, identifique mercados grandes onde o software é odiado pelos usuários. Então veja se algum intersecta com seu conhecimento
