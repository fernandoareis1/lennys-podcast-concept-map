# Plataforma por Trajetória de Workflow

**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Dylan Field]]

## Ideia central
A estratégia de plataforma da Figma: **trace o workflow completo do usuário e identifique os casos de uso que, se incluídos no produto principal, adicionariam complexidade desnecessária** — e extraia-os em superfícies dedicadas.

**O padrão observado:**
Usuários usavam o Figma Design para whiteboarding → adicionaria complexidade ao produto principal → extraiu FigJam. Usavam para slides → extraiu Figma Slides. Para handoff dev → Dev Mode. Para brand assets em bulk → Buzz. Para publicar sites → Figma Sites. Para prototipagem rápida de apps → Make.

**A lógica de "um para dois" vs. "dois para N":**
Ir de 1 produto para 2 é muito difícil. Ir de 2 para N é muito mais fácil. O primeiro produto secundário (FigJam) prova que você consegue expandir — depois o modelo escala.

**Por que não jogar tudo no produto principal:**
- Adicionar um caso de uso especializado ao produto principal cria complexidade para todos
- Marketing quer rapidez + compliance de brand → pack no Figma Design tornaria complexo para designers
- Solução: superfície dedicada por workflow → cada produto pode ser otimizado para seu público específico

**Como FigJam surgiu:**
- Dylan tinha intuição sobre o whiteboarding há anos, sem conseguir articular o "por quê agora"
- COVID fez o feedback explodir — validação externa deu convicção para agir
- Design sprint: 20 ideias em um dia, algumas se tornaram definitivas (Cursor Chat, fun as differentiator)

## Como aplicar
1. Mapeie todos os casos de uso que existem no seu produto principal mas não foram priorizados para esse público
2. Para cada um, pergunte: "Seria melhor como produto dedicado com seu próprio conjunto de decisões de UX?"
3. Use o workflow do usuário como fio condutor: onde eles querem ir depois de cada passo?
4. O critério de extração: se colocar no produto principal aumenta complexidade para o público original sem adicionar valor proporcional → candidate para nova superfície
