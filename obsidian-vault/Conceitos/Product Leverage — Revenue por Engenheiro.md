# Product Leverage — Revenue por Engenheiro
**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Insight · **Fontes:** [[Nabeel S. Qureshi]]

## Ideia central
A Palantir usava **revenue per engineer** como North Star para medir se estava sendo uma empresa de produto ou uma consultoria disfarçada. Quando cada novo cliente exigia jogar muita gente nele, o número ia para baixo — sinal de que estava construindo bespoke. Quando o produto existente servia o novo cliente sem novas contratações, o número ia para cima — isso é product leverage. O objetivo é que cada novo cliente encontre valor no produto já existente, não em soluções criadas do zero só para eles.

## Como aplicar
1. **Monitore revenue por headcount de produto**: crescendo revenue enquanto headcount cresce mais lentamente = produto escalando. Crescendo ambos proporcionalmente = consultoria.
2. **Diferencie FDE temporário de consultoria permanente**: a FDE cria produto que abstrai e serve futuros clientes. Consultoria cria solução que só serve aquele cliente e exige manutenção dedicada.
3. **Pergunte a cada customização**: "Este trabalho vira produto ou só serve este cliente?" Se a resposta for "só este cliente", é consultoria — mesmo que o cliente pague muito.
4. **Use como filtro de roadmap**: priorize features que reduzem o esforço por cliente novo. Cada feature que elimina customização aumenta o leverage.

## Insights por autor
### [[Nabeel S. Qureshi]]
- "Revenue per engineer era o North Star. Quanto mais product leverage, mais alto esse número. Se você tem que jogar muitas pessoas em cada problema marginal, você está em efeito sendo uma consultoria."
- O dilema: quando o modelo funciona, você pode servir um novo cliente sem contratar mais ninguém. Quando não funciona, você está reescrevendo código do zero para cada cliente.
- A Palantir resolveu o dilema gradualmente: de Jupyter Notebooks e integração primitiva para o Foundry — que qualquer cliente consegue usar com uma fração do suporte humano.
