# Product-Led Sales (PLS)

**Fluxo:** [[06 - Growth & Aquisição]] · **Tema:** [[Growth — Modelos de crescimento]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Elena Verna]]

## Ideia central
No PLS, o **produto** (não o marketing) adquire e ativa usuários individuais; o produto também cria o pipeline de vendas, e então **vendas fecha** contratos enterprise. A diferença do modelo tradicional: product owes pipeline accountability — não é marketing gerando leads para vendas.

A escalada é de três camadas: problema individual → problema de time → problema enterprise. Só vale PLS quando o deal size justifica (contratos >US$15K; tipicamente 200+ colaboradores).

## Como aplicar
1. Espere **hand-raisers orgânicos** (usuários pedindo upgrade enterprise via suporte) antes de contratar qualquer SDR.
2. Certifique-se de que o produto consegue **self-serve ativar** o usuário individual (sem isso não há PLG base).
3. Defina qual é o **problema enterprise** (≠ problema individual) — a proposta de valor muda de camada.
4. Use [[PQA — Product-Qualified Account]] como sinal de handoff para vendas (não MQL clássico).
5. Evite o erro de jogar PLS no marketing — o produto precisa ser accountable pelo pipeline.

## Insights por autor
### [[Elena Verna]]
- "Em product-led sales, product adquire e ativa o cliente, e product cria pipeline para vendas — não é marketing criando pipeline para vendas."
- A pior combinação: tentar fazer PLS dentro de uma equipe de marketing. "Recipe for disaster — você vai estar no failure mode em seis meses."
- Três canais de lead: (1) usuário com uso já é o lead, (2) usage existe mas o lead ainda não (marketing traz), (3) sem uso nenhum (top-down clássico). Cada canal tem playbook e taxa de conversão diferentes.
