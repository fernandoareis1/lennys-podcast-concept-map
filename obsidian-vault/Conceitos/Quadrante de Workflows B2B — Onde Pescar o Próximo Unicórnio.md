# Quadrante de Workflows B2B — Onde Pescar o Próximo Unicórnio
**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Framework · **Fonte:** [[Oji Udezue]]

## Ideia central
Antes de construir, escolha o problema certo. Problemas de workflow B2B têm poder preditivo de sucesso. O framework usa duas dimensões para mapear onde os maiores negócios prosperam:

**Dimensão 1 — Amplitude** (para quantos departamentos o workflow se aplica?):
- **Todos** (everyone): workflow usado do porteiro ao CEO — email, calendário, colaboração, escrita
- **Niche**: workflow específico de um departamento — recrutadores, vendedores, engenheiros, financeiro

**Dimensão 2 — Frequência** (com que frequência o workflow é executado?):
- **Alta frequência**: diariamente ou múltiplas vezes por semana
- **Baixa frequência**: semanalmente ou menos

**Os quatro quadrantes:**

| | Alta Frequência | Baixa Frequência |
|---|---|---|
| **Todos** | Domínio de Microsoft/Google — email, Word, Excel. Muito difícil de entrar. | Expense reports, forms. Difícil de monetizar, exige expansão. |
| **Niche** | ★ **Hi-Ni — O sweet spot do B2B SaaS**. Jira, Salesforce, HubSpot. | Planejamento financeiro (FP&A), ERP. Possível, mas exige estratégia de expansão. |

**Hi-Ni (High Frequency Niche) é o sweet spot:**
- Departamentos específicos consideram o workflow missão-crítica
- Alta frequência = baixo churn, alto valor percebido, WoM natural
- Exemplos: Jira (engenharia), Salesforce (vendas), HubSpot (marketing), Calendly (quem agenda externamente)
- Atlassian nasceu em Hi-Ni e está expandindo para "everyone" com Confluence

**O framework não é estático:**
- Estratégia = navegar de quadrantes menores para maiores
- Typeform é low-frequency everyone → Oji quer mover para Hi-Ni para marketers e salespeople
- Uma estratégia de expansão: encontre o subconjunto de usuários que considera seu produto missão-crítica → aprofunde para eles → expanda a partir daí

## Como aplicar
1. Mapeie onde sua solução cai no quadrante: qual departamento? Com que frequência?
2. Identifique se existe uma persona para quem seu produto é Hi-Ni (mesmo que para a maioria seja low-frequency)
3. Aprofunde a solução para essa persona Hi-Ni antes de expandir para outras
4. Se você está em low-frequency niche: adicione um módulo que algum subgrupo use diariamente (ex: expense report que o financeiro usa todo dia)
5. Pré-validação: se você fosse um VC, esse quadrante justifica o investimento?
