# Quadrante de Workflows B2B — Onde Pescar um Unicórnio

**Fluxo:** [[02 - Estratégia & Visão]] · **Tema:** [[Estratégia — Estratégia de produto]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Oji Udezue]]

## Ideia central
Todo produto B2B pode ser posicionado em um quadrante de duas dimensões: **amplitude** (todo mundo usa × nicho específico) e **frequência** (uso diário/semanal × uso ocasional). A combinação dessas duas dimensões determina o potencial de crescimento e o risco competitivo de cada categoria.

```
                  NICHO              TODO MUNDO
                ┌──────────────┬──────────────────────┐
  ALTA          │  SWEET SPOT  │  Dominado por        │
  FREQUÊNCIA    │  "high-ni"   │  Microsoft/Google/   │
                │  Jira, CRM,  │  Atlassian           │
                │  martech     │                      │
                ├──────────────┼──────────────────────┤
  BAIXA         │  Pequeno     │  Oportunidades       │
  FREQUÊNCIA    │  demais      │  especializadas de   │
                │              │  baixo RPC            │
                └──────────────┴──────────────────────┘
```

## O sweet spot: "high-ni"
O quadrante de **alta frequência + nicho específico** ("high-ni") é onde os unicórnios B2B nascem. Três razões:

1. **Frequência cria hábito e retenção**: uso diário ou semanal gera dependência funcional e alto custo de troca
2. **Nicho permite diferenciação profunda**: ao contrário de "todo mundo", nicho permite otimização para o workflow específico sem satisfazer denominadores comuns
3. **Evita colisão direta com big tech**: Microsoft, Google e Atlassian dominam as colunas "todo mundo"; a coluna "nicho" é território mais livre

Exemplos no sweet spot: Jira (engenharia de software), Salesforce (equipes de vendas), HubSpot (marketing), Workday (RH). Todos: nicho de profissão + alta frequência de uso.

## O erro de estratégia mais comum
Começar no quadrante "todo mundo + baixa frequência" — product-market fit é difícil porque você compete com Excel, email, e o status quo implícito. Também é tentador por parecer "grande mercado".

O segundo erro é construir para "todo mundo + alta frequência": esse quadrante só é acessível a empresas com bilhões em distribuição (Microsoft Teams, Google Workspace, Slack via viralidade pré-escala).

## Como o Calendly executou esse mapa
O Calendly começou no quadrante errado: **baixa frequência + todo mundo** (qualquer pessoa que marca reuniões ocasionalmente). Cresceu por viralidade (link de agendamento expõe o produto), mas o teto de ARPU era baixo.

O movimento estratégico foi migrar para "high-ni": **alta frequência + nichos específicos** — marketers que fazem demos, salespeople com pipeline intenso, recruiters com volume de entrevistas. Nesses nichos, agendamento não é tarefa ocasional: é core do workflow. Pricing pode subir; NPS sobe junto.

## Como usar o framework
1. Mapeie onde você está hoje nos dois eixos (honestamente)
2. Identifique o quadrante de destino e o que mudaria no produto/GTM para chegar lá
3. Defina o sequenciamento: nicho estreito com alta frequência → expansão horizontal ou vertical

O framework é também um filtro de oportunidade: se uma categoria nova aparece, pergunte imediatamente em qual quadrante ela vive.
