# Razão Decisiva Única (SDR)
**Fluxo:** [[09 - Liderança & Times]] · **Tema:** [[Liderança — Colaboração & comunicação]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Rahul Vohra]]

## Ideia central
SDR (Single Decisive Reason) é a exigência de que toda decisão importante possa ser justificada por **uma única razão que, sozinha, é suficiente**. Se você precisa de três, cinco ou dez razões menores para defender uma decisão, nenhuma delas é forte o bastante. Uma coleção de razões fracas raramente soma uma razão forte — mas cria a ilusão de que sim.

O insight central: o custo cognitivo de listar muitos prós e contras **distorce** o processo. Em grupo, o volume de contras (mesmo fracos) tende a paralisar por excesso de prudência. O SDR corta esse ruído.

## Como aplicar
1. **Na hora da decisão:** antes de listar prós/contras, pergunte: "Qual é a razão decisiva única para fazer isso?"
2. **Teste de isolamento:** "Se apenas isso fosse verdade — e todo o resto não fosse — eu ainda faria?" Se sim, é o SDR. Se não, continue procurando
3. **Quando alguém apresenta múltiplas razões:** pergunte "Se só uma dessas fosse verdade, qual você escolheria?" Incapacidade de responder sinaliza que a decisão ainda não está madura
4. **Custo de oportunidade como filtro:** toda feature construída é outra não construída. Um SDR forte justifica a escolha mesmo diante desse custo; uma lista fraca não
5. **Aplicação recursiva:** o SDR pode ser aplicado em cascata — inclusive ao próprio framework ("qual é o SDR para usar SDR?")

## Insights por autor
### [[Rahul Vohra]]
- "For important decisions, you should be able to identify one reason that on its own supports the decision."
- "A collection of weak reasons rarely adds up to a high quality reason to do something."
- "If this was true and all the other things were not true, would I still do it? That's the bar for a good decision."
- "Any decision has an opportunity cost. If we build for weak reasons whereas we could build something else for one strong reason, I'd much rather build that."
