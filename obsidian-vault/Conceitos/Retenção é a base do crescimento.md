# Retenção é a base do crescimento

**Fluxo:** [[07 - Ativação & Retenção]] · **Tema:** [[Retenção — Retenção & churn]] · **Camada:** L3
**Tipo:** Mindset
**Fontes:** [[Elena Verna]], [[Albert Cheng]]

## Ideia central
Sem **retenção** não existe PMF — e sem PMF, growth é inútil. Só faz sentido investir em crescimento quando os clientes ficam (e diriam "não consigo viver sem este produto").

## Insights por autor
### [[Elena Verna]]
- Pré-requisitos para montar time de growth: (1) PMF com boa retenção; (2) dados suficientes.
- Growth **amplifica** PMF existente; não cria PMF nem reverte declínio.

> 🎧 [Elena Verna (10:00–12:00)](https://www.youtube.com/watch?v=IHwS2By9UKM)

### [[Albert Cheng]]
- D1 retention de 30–40% é sólida para apps consumer; abaixo disso, matematicamente difícil construir uma base de usuários ativos grande o suficiente.
- Em produtos maduros: ~80% dos usuários ativos diários já são usuários existentes ou ressuscitados, não novos. Resurrected users = grande alavanca negligenciada.
- Exceção: Grammarly (ferramenta passiva que não exige abertura proativa) — para esse tipo de produto, o momento de ativação/instalação importa mais que a retenção diária.
- "Retenção é ouro para empresas de assinatura consumer. Sem reter usuários, a pressão toda vai para fazer eles pagarem no dia 1 — isso é muito difícil."
