# Roadmap Secreto — Criar o Que Ninguém Pediu

**Fluxo:** [[03 - Priorização & Roadmap]] · **Tema:** [[Priorização — Priorização & foco]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Gaurav Misra]]

## Ideia central
Todo time de produto tem um **roadmap público** — a lista de features que usuários pedem. O problema: se os usuários pedem para você, estão pedindo para todo mundo. Todos os concorrentes têm basicamente a mesma lista, e o diferencial máximo é priorização ou execução ligeiramente melhor. Os **maiores game-changers vêm de um roadmap secreto** — coisas que ninguém pediu, que o usuário poderia inicialmente rejeitar, mas que, uma vez testadas, mudam o comportamento para sempre.

## Os dois roadmaps da Captions
| Roadmap Público | Roadmap Secreto |
|---|---|
| Vem de feedback de usuários | Vem de brainstorm interno |
| Todos os concorrentes têm | Ninguém conhece até o lançamento |
| Melhoria incremental | Mudança de comportamento |
| Diferencial: melhor execução | Diferencial: insight único de usuário |
| Wins pequenos a médios | Wins grandes |

## Como gerar ideias para o roadmap secreto
1. Brainstorm trimestral toda a empresa (não só produto) — engenharia, design, marketing, recrutamento, todos incluídos
2. Votar e ranquear ideias coletadas
3. Time de produto filtra por viabilidade técnica e potencial de impacto
4. Nunca divulgar publicamente antes do lançamento

## Exemplo: Eye Contact
Captions foi a primeira empresa a lançar correção de olhar (redirecionar os olhos para a câmera mesmo quando o criador está olhando para o teleprompter). Ideia surgiu do brainstorm interno; Nvidia entregou a tecnologia porque não sabia para que seria usada. O vídeo viral demonstrando a feature foi repostado em todos os idiomas e ainda acumula milhões de views.

## Por que funciona
- Usuários só pedem o que já conhecem
- O papel do produto é mostrar o que eles não sabiam que precisavam
- Uma vez que usam, nunca voltam atrás — esse é o critério para incluir na secret list
- "That's what a product is, that's success"
