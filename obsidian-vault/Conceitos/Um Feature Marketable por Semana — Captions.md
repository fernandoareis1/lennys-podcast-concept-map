# Um Feature Marketable por Semana — Captions

**Fluxo:** [[09 - Liderança & Times]] · **Tema:** [[Liderança — Modelos de time]] · **Camada:** L3
**Tipo:** Framework · **Fontes:** [[Gaurav Misra]]

## Ideia central
Meta de execução da Captions: **cada engenheiro entrega um feature marketable toda semana**. "Marketable" tem definição precisa — é o feature pelo qual um usuário poderia assinar o app ou instalar o app sozinho. Não é uma feature de table stakes que todo mundo tem: é algo que você pode apontar e dizer "venha ao nosso app só por isso."

## O que é "marketable"
- Alguém pagaria para usar o app *só por esse feature*
- Pode ser demonstrado em um post de redes sociais
- É único — não apenas uma feature que já existe em todo concorrente
- NÃO são features de table stakes (ex: justify alignment num editor de texto — ninguém baixa um app por isso)

## Como manter qualidade com velocidade semanal
A armadilha é cortar qualidade quando o prazo aperta. O método correto é **cortar escopo, não qualidade**:
1. Liste cada elemento que levará tempo para construir
2. Pergunte: "e se removermos isso — o produto ainda é útil?"
3. Repita até chegar no mínimo que ainda é genuinamente útil
4. Lance esse núcleo. Usuários vão reclamar do que falta → essa é a lista para a semana seguinte

## Por que funciona
- **Feedback de reclamação = PMF**: se ninguém reclama, é red flag (não importa o suficiente)
- Ciclos semanais criam volume de hipóteses testadas; o que funciona recebe mais esforço
- Mantém engajamento da equipe: todo mundo tem algo para mostrar toda semana
- Mantém o produto acima do ruído: lançamentos frequentes são conteúdo constante de marketing

## Exceções e infraestrutura
- Q4 = "infrastructure quarter": dívida técnica acumulada é paga nesse período
- Trabalho de back-end que não vira feature não entra no contador semanal, mas tem slot dedicado no calendário anual

## Relação com Intensidade por Minuto
Conceito análogo ao de [[Intensidade por Minuto — Kilojoules por Hora]] de [[Farhan Thawar]] (Shopify): o objetivo não é mais horas mas mais resultado por unidade de tempo. Captions alcança isso via cadência semanal forçada; Shopify via GSD updates + pair programming + six-week reviews.
