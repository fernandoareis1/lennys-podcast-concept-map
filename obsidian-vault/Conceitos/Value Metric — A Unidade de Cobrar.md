# Value Metric — A Unidade de Cobrar

**Fluxo:** [[08 - Monetização & Pricing]] · **Tema:** [[Monetização — Modelos & poder de pricing]] · **Camada:** L3
**Tipo:** framework · **Fontes:** [[Naomi Ionita]]

## Ideia central
A **value metric** é a unidade de valor que o cliente deriva do produto — e que deve ser a base da precificação. Cobrar por seats (modelo histórico de SaaS) cria desalinhamento com o valor real. Cobrar pela value metric cria um **escalonador natural**: quanto mais o cliente usa e ganha valor, mais ele paga — gerando receita crescente sem necessidade de forçar upgrades.

## Como aplicar
1. Identifique a pergunta: "O que torna o produto mais valioso para o cliente à medida que ele cresce?" (ex.: mensagens enviadas, invoices criadas, TB de storage, chamadas de API, palavras geradas).
2. Valide a value metric com pesquisa de usuários: a métrica deve ser compreensível ("posso prever minha conta?") e correlacionada ao valor percebido.
3. Praticamente nenhuma empresa deve ser 100% usage-based ou 100% seat-based — o modelo híbrido domina: planos fixos com limites de consumo da value metric.
4. Use os limites de quota como gatilho de upgrade: usuário que excede o limite de invoices/mês no plano básico tem proposta clara de upgrade.
5. Considere o problema do CFO: modelos puramente variáveis dificultam orçamento e podem gerar surpresas desagradáveis que prejudicam a retenção.

## Insights por autor
### [[Naomi Ionita]]
- No Evernote: single flat fee ($45/ano) sem value metric → uma única conversão, sem escalonamento natural.
- No Invoice2go: number of invoices como value metric → criou quota limits por plano e loop de upsell ao atingir limites.
- "Matching price to value creates alignment with your user and a natural escalator as they use it more."
- Apenas ~5% das empresas de SaaS têm modelos 100% usage-based; a grande maioria usa hybrid.
